# 5.8 继承的设计技巧
+ `将公共操作和域放在父类`  
+ `不要使用受保护的域`  
    - 子类集合是无限制的,任何人都能够由某个类派生一个子类,
    并编写代码以直接放分protected的实例类,从而破坏封装性  
    - 在Java程序设计语言中,在同一个包中的所有类都可以访问protected域,
    不管是否为这个类的子类  
+ `使用继承实现"is-a"关系`  
+ `除非所有继承的方法都有意义`,否则不要使用继承  
+ 在覆盖方法时,`不要改变预期的行为`  
+ `使用多态,而非类型信息`  
+ `不要过多地使用反射`
