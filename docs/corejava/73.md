# 7.3 使用异常机制的技巧  
* 异常处理不能代替简单测试  
    * 使用异常的基本规则:`只在异常情况下使用异常机制`  
* 不要过分的细化异常  
* 利用异常层次结构  
    * 不要只抛出RuntimeException异常.应该寻找更加适当的子类或创建自己的异常类  
    * 不要只捕获Throwable异常,否则,会使程序代码更难读,更难维护
    * 考虑受查异常和非受查异常的区别  
    * 将一种异常转换成另一种更加适合的异常时不要犹豫    
* 不要压制异常  
* `早抛出,晚捕获`  
    * 在检测错误时,"苛刻"要比放任更好  
    * 不要羞于传递异常
